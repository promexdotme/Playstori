Installation

Actually there is no installation proccess for HTML5 content, you can just double-click .html file to open it.
But there are a restriction for local HTML file, browser can't load local files, so this restriction causing all game assets like images and sounds can't be loaded and shown.

You can fix it by opening the game (html) with localserver.
Local server apps: Laragon, NodeJS, USBWebserver (Light weight) and there are many other similar software.

If you upload the game files on live webserver (Your website), it should run fine without any technical issues.

Item Structure

As you see, there is 2 main folder inside item file, "cloudarcade" and "html5" folder.

cloudarcade folder mean, the game inside this folder is suitable for CloudArcade CMS or this game version is already integrated with CloudArcade API.

html5 folder mean, is a clean version of the game or the main game files, this version is not contain any external API and can be used as your base project.

Both these 2 folder have 2 folder inside, "minified" and "source", what it's mean ? "minified" mean, all the scripts inside this folder is minified or obsfucated, simply the script is protected and very hard to read, so anyone who play your game and inspect it can't copy or stole the game code or make it harder for anyone who trying to hack or crack the game.
This "minified" version is strongly recommended to be published online (public).

"source" or also called the source code, the game scripts on this version is readable and this version is used for base project, example if you want to edit game scripts or modify the game.

Game scripts

This section is refering game scripts inside "scripts" folder

Each scene of the game is separated by script file

File list:

  1. phaser.min.js
  2. boot.js
  3. preload.js
  4. menu.js
  5. game.js

phaser.min.js

This game is using Phaser 3 framework, and this is the framework script/file.

boot.js

boot.js or "Boot" scene loaded initially when the game opened, boot.js purpose is to load initial files (background, game title and start button) that will be shown on the "preload" scene.

preload.js

After "Boot" scene that load initial files finished, scene will be moved to "Preload", all game files will be loaded here also it's show loading progress, background, and game title that has been loaded from "Boot".

menu.js

After all game files has been loaded from "Preload" scene, start button will be shown (If exist). if start button clicked, scene will be moved to "Menu" (menu.js)

game.js

All gameplay related script is stored here, this script handle all logic from the gameplay.

If you scroll to the bottom of game.js file, you will see "config" variable that used for Phaser configuration

In the bottom of "config" variable, you will see

var game = new Phaser.Game(config);

This is where the game script begin, without this script line, the game script will never be started and just show empty/blank page.
